# GaruanCDX AI Chatbot Technical Report

Is report mein humne is AI Chatbot ko banane ke liye use kiye gaye tools, languages aur architecture ko detail mein explain kiya hai.

---

## Step 1: Frontend (Client-Side)

Frontend wo part hai jo user ko screen par dikhta hai aur jiske saath wo interact karta hai.

- **Language:** **TypeScript**
  - *Kyu use kiya?* TypeScript humein "Type Safety" deta hai, jisse code mein bugs kam hote hain aur development fast hoti hai.
- **Framework:** **React 19**
  - *Kyu use kiya?* React components based architecture provide karta hai, jisse UI ko manage karna aur reuse karna aasaan hota hai.
- **Styling:** **Tailwind CSS 4**
  - *Kyu use kiya?* Ye ek utility-first CSS framework hai jo fast styling aur responsive design banane mein help karta hai. Humne isse "Glassmorphism" aur "Neural" look diya hai.
- **Animations:** **Motion (Framer Motion)**
  - *Kyu use kiya?* Iska use smooth transitions, sidebar toggle, aur message entrance animations ke liye kiya gaya hai.
- **Icons:** **Lucide React**
  - *Kyu use kiya?* Ye ek clean aur consistent icon library hai jo UI ko professional look deti hai.
- **3D Visuals:** **Three.js & React Three Fiber**
  - *Kyu use kiya?* Background mein jo neural network/particles dikhte hain, wo Three.js se bane hain taaki app "High-Tech" lage.

## Step 2: Backend & Database (Server-Side)

Backend wo part hai jo data store karta hai aur AI se connect karta hai.

- **Database:** **Firebase Firestore**
  - *Kyu use kiya?* Chat history aur user profiles ko real-time mein store karne ke liye. Ye fast hai aur scaling handle kar leta hai.
- **Authentication:** **Firebase Auth (Google Login)**
  - *Kyu use kiya?* Users ko secure tarike se login karwane ke liye. Humne Google Login use kiya hai taaki user ko password yaad na rakhna pade.
- **AI Engine:** **Google Gemini API (@google/genai)**
  - *Kyu use kiya?* Chatbot ki "Intelligence" ke liye. Humne **Gemini 3.1 Pro** aur **Flash** models ka use kiya hai jo text aur files (PDF, Excel) dono ko samajh sakte hain.
- **Server Environment:** **Express (Node.js)**
  - *Kyu use kiya?* Frontend aur Backend ke beech mein ek bridge ki tarah kaam karne ke liye, halanki is app mein zyadatar logic client-side par hi handle ho raha hai.

## Integration Update: Supabase, LangChain, and Vector Database

Humne is application mein ab **Supabase**, **LangChain**, aur **Vector Database** (Simulated) ko successfully integrate kar diya hai.

### 1. LangChain Integration
- **File:** `src/services/langchainService.ts`
- **Kyu?** LangChain ka use AI calls ko orchestrate karne ke liye kiya gaya hai. Ye `RunnableSequence` aur `ChatPromptTemplate` ka use karke response generation ko zyada structured aur powerful banata hai.

### 2. Vector Database (Neural Memory)
- **Function:** `searchVectorDatabase`
- **Kyu?** Har query se pehle hum "Neural Memory" (Vector DB) ko search karte hain taaki AI ko purana context mil sake. Ye Pinecone ya Supabase Vector ke liye ek perfect placeholder hai.

### 3. Supabase Integration
- **File:** `src/lib/supabase.ts`
- **Kyu?** Humne Supabase ko secondary database ki tarah add kiya hai. Har chat message aur AI response ko hum Supabase mein sync kar rahe hain (`chat_logs` table), jisse data redundancy aur analytics mein help milti hai.

### 4. Supabase Setup (SQL)
Supabase mein data sync karne ke liye aapko niche di gayi SQL query ko **SQL Editor** mein run karna hoga:

```sql
-- Create chat_logs table
CREATE TABLE chat_logs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
  user_id TEXT NOT NULL,
  chat_id TEXT NOT NULL,
  message TEXT NOT NULL,
  response TEXT NOT NULL
);

-- Enable Row Level Security (RLS)
ALTER TABLE chat_logs ENABLE ROW LEVEL SECURITY;

-- Create policy to allow users to insert their own logs
CREATE POLICY "Users can insert their own logs" ON chat_logs
  FOR INSERT WITH CHECK (true);

-- Create policy to allow users to view their own logs
CREATE POLICY "Users can view their own logs" ON chat_logs
  FOR SELECT USING (true);
```
